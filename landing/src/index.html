Pricing

Fees are straight forward and visible up front before starting the transaction. There are no extra hidden costs.

They are also situational, fees are only taken when the operation is mathematically incogruent and would have a remaining, so for most of the scenarios, no fees will be taken.

Here you can find how the fee system works.

No fees
Transactions that can easily be split between users and don't have a remainder are free.
10 / 4 = 2.5
15 / 3 = 5
500 / 4 = 125
Fees
Transactions that can not be divided will have the remaining in the cents point taken as fee
10 / 3 = 3.33333...
100 / 7 = 14.285714...
500 / 9 = 55.55555...
This numbers can not be divided so, to simplify the operation, the following amount will be extracted from the transaction
0.0001 will be taken as fee
0.0005 will be taken as fee
0.0005 will be taken as fee
And each user will receive

14.2857
55.5555

Integrate it into your own contract

Easily implement it into your own smart contract to automatically distribute payments between different acounts.

The system supports any ERC-20 token. Find more about it in the documentation. 

            
interface I1Transfer {
  function splitPayment(address[] memory recipients) 
    external payable;
            
  function splitTokenPayment(
    address[] memory recipients,
    uint256 amount,
    address token
    ) external;
}
